### 问题分析
当前添加标签后界面不刷新的问题是因为 `FileTagRepositoryImpl.observeRootTags()` 方法使用了简单的 `flow` 构建器，它只在首次订阅时发射一次数据，然后就完成了，不会在标签数据变化时自动更新。而数据库实现使用了 Room 的 Flow，它会自动监听数据库变化并触发更新。

### 修复方案
使用 `StateFlow` 结合标签变化通知机制，确保标签数据变化时 UI 能够及时刷新：

1. **修改 `TagFileManager.kt`**：
   - 添加标签变化通知机制（使用 `MutableSharedFlow`）
   - 在所有修改标签的方法中触发通知

2. **修改 `FileTagRepositoryImpl.kt`**：
   - 使用 `StateFlow` 维护当前标签状态
   - 监听标签变化通知，更新 `StateFlow`
   - `observeRootTags()` 方法返回 `StateFlow`

3. **修改相关方法**：
   - 在 `createTag`、`deleteTag`、`renameTag` 等所有修改标签的方法中触发标签变化通知

### 具体实现步骤
1. 在 `TagFileManager.kt` 中添加 `tagChanges` 共享流
2. 在所有修改标签的方法（`writeTag`、`deleteTag` 等）中调用 `tagChanges.emit(Unit)`
3. 在 `FileTagRepositoryImpl.kt` 中创建 `rootTagsStateFlow`
4. 在 `FileTagRepositoryImpl.kt` 中监听 `tagChanges` 并更新 `rootTagsStateFlow`
5. 修改 `observeRootTags()` 方法返回 `rootTagsStateFlow`

### 预期效果
- 添加标签后，界面会立即刷新，显示新添加的标签
- 删除、重命名标签后，界面会立即更新
- 标签引用关系变化后，界面会立即反映变化
- 保持与原数据库实现相同的 API 和行为，确保无缝迁移

### 技术要点
- 使用 `StateFlow` 保持当前标签状态
- 使用 `SharedFlow` 作为事件总线，传递标签变化通知
- 确保线程安全，所有文件操作在后台线程执行
- 保持代码的可维护性和扩展性
